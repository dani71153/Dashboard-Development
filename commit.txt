feat(control): Mejoras robustas en control manual - tópico y frecuencia configurables, soporte TwistStamped

- Ahora los comandos de movimiento se publican en cualquier tópico configurable desde el dashboard (por defecto `/diff_drive_controller/cmd_vel`), sin editar el JS.
- El tipo de mensaje es `geometry_msgs/TwistStamped`, compatible con controladores que requieren encabezado y tiempo.
- Soporta edición y sincronización de sliders y cajas de texto para velocidades lineal y angular; valores limitados y siempre consistentes.
- Permite elegir la frecuencia de publicación (1–30 Hz) para ajustar a hardware real y evitar saturar ROS.
- Estado de publicación visible: muestra tópico, frecuencia y valores actuales enviados.
- Todos los controles y entradas se bloquean automáticamente si ROS se desconecta, previniendo comandos inválidos.
- Soporte para teclas rápidas WASD y espacio para control directo desde el teclado.
- Modularidad: código preparado para cambiar fácilmente a otros tipos de mensaje si algún robot lo requiere.

feat(nodos): Visualización y consulta avanzada de nodos y tópicos ROS con comandos dinámicos

- Ahora el dashboard muestra todos los tópicos y nodos en columnas separadas, con actualización manual o automática al cambiar de pestaña.
- Agregados dropdowns para seleccionar cualquier nodo o tópico y consultar info detallada usando los comandos de ROSBridge (`info`, `publishers`, `subscribers`, `node_topics`, `topic_type`).
- Resultados de las consultas aparecen en tiempo real, sin recargar la página.
- El dashboard deshabilita o limpia las listas y los comandos si ROS está desconectado o no responde.
- Errores y estados se muestran de forma clara y explícita para mejor UX.
- Código estructurado para fácil extensión y mantenimiento.
